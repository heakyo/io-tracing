## 问题描述

给定一个链表的头节点 `head` 和一个非负整数 `k`，将链表向右旋转 `k` 个位置。

所谓"向右旋转 k 个位置"，就是把链表末尾的 k 个节点搬到链表头部，其余节点的
相对顺序保持不变。

**约束条件：**

- 链表中节点的数量在范围 `[0, 500]` 内
- `-100 <= Node.val <= 100`
- `0 <= k <= 2 * 10^9`

**示例：**

```
输入：head = [1, 2, 3, 4, 5], k = 2
输出：[4, 5, 1, 2, 3]
解释：向右旋转 1 步后 [5, 1, 2, 3, 4]，再旋转 1 步后 [4, 5, 1, 2, 3]。

输入：head = [0, 1, 2], k = 4
输出：[2, 0, 1]
解释：k=4 对长度 3 取模得 1，等价于向右旋转 1 步。
```

## 核心思路

> 想象一条项链：把链子从某个位置剪开，然后把后半段接到前半段前面，就完成了旋转。
> 关键是找对剪开的位置。

这就是**计数 + 取模 + 分割重连**的核心直觉。

具体做法：

1. **计数：** 遍历整个链表，统计节点总数 `size`，同时用 `prev` 记录尾节点
2. **取模：** 计算 `n = k % size`，消除多余的完整轮旋转。若 `n == 0` 说明旋转后链表不变，直接返回
3. **定位分割点：** 从头节点出发，向前走 `size - n - 1` 步，到达分割点 `cur`。此时 `cur` 是新链表尾部的最后一个节点
4. **分割重连：** 保存 `cur->next` 为新头节点 `next`，断开 `cur->next = NULL`，将原尾节点 `prev->next` 指向原头节点 `head`，完成环形重连后从 `next` 处断开

```c
struct ListNode* rotateRight(struct ListNode* head, int k)
{
	struct ListNode *cur, *next, *prev;
	struct ListNode dummy = {0, head};
	int size, n, i;

	if (k == 0 || head == NULL)
		return head;

	dummy.next = head;
	prev = &dummy;
	cur = head;
	size = 0;

	while (cur) {
		size++;
		prev = cur;
		cur = cur->next;
	}

	n = k % size;
	if (n == 0)
		return head;

	cur = head;
	for (i = 0; i < size - n - 1; i++)
		cur = cur->next;

	next = cur->next;
	cur->next = NULL;
	prev->next = head;
	dummy.next = next;

	return dummy.next;
}
```

关键不变量：分割点位于第 `size - n` 个节点之前，即从头数第 `size - n - 1` 个节点（0 起）。分割后，后 `n` 个节点成为新链表的前 `n` 个节点。

## 逐步推演

以 `[1, 2, 3, 4, 5]`，`k = 2` 为例。

链表结构：

```
1 -> 2 -> 3 -> 4 -> 5 -> NULL
```

目标：向右旋转 2 位，结果为 `[4, 5, 1, 2, 3]`。

**阶段零：边界检查**

- `k = 2`，不为 0；`head` 不为 NULL。继续执行。

**阶段一：遍历计数，找到尾节点**

初始状态：`prev = &dummy`，`cur = head`（节点 1），`size = 0`。

| 轮次 | cur 指向 | size 变化 | prev 移动后 | cur 移动后 |
|:----:|:--------:|:---------:|:-----------:|:----------:|
| 1 | 节点 1 | 1 | 节点 1 | 节点 2 |
| 2 | 节点 2 | 2 | 节点 2 | 节点 3 |
| 3 | 节点 3 | 3 | 节点 3 | 节点 4 |
| 4 | 节点 4 | 4 | 节点 4 | 节点 5 |
| 5 | 节点 5 | 5 | 节点 5 | NULL |

循环结束：`size = 5`，`prev` 指向节点 5（尾节点），`cur = NULL`。

**阶段二：取模**

```
n = k % size = 2 % 5 = 2
```

`n = 2`，不为 0，继续执行。

**阶段三：定位分割点**

需要走 `size - n - 1 = 5 - 2 - 1 = 2` 步。`cur` 从节点 1 出发。

| 步骤 | i 值 | cur 移动后 |
|:----:|:----:|:----------:|
| 1 | 0 | 节点 2 |
| 2 | 1 | 节点 3 |

循环结束：`cur` 指向节点 3，这就是分割点。

**阶段四：分割重连**

```
next = cur->next          --> next 指向节点 4（新头节点）
cur->next = NULL          --> 节点 3 ->next = NULL（断开）
prev->next = head         --> 节点 5 ->next = 节点 1（尾接原头）
dummy.next = next         --> dummy.next = 节点 4
```

操作后的链表：

```
4 -> 5 -> 1 -> 2 -> 3 -> NULL
```

返回 `dummy.next` = 节点 4，结果正确。

## 易错点分析

### (a) k 大于链表长度时的取模处理

`k` 的取值范围可达 `2 * 10^9`，远远超过链表长度。旋转 `size` 次等于没有旋转，
因此有效旋转次数为 `n = k % size`。遗漏这一步会导致 `for` 循环越界访问空指针。

同时需要注意取模结果为 0 的情况：`n == 0` 表示旋转后链表不变，应直接返回原头节点。
代码中正确处理了这一点：

```c
n = k % size;
if (n == 0)
    return head;
```

### (b) dummy 节点是多余的

当前实现声明了 `struct ListNode dummy = {0, head}`，并在最后通过 `dummy.next`
返回新头节点。但仔细观察代码逻辑，`dummy.next` 的值在最后一步被赋值为 `next`，
而 `next` 就是新的头节点。因此完全可以直接 `return next`，不需要 `dummy` 节点：

```c
next = cur->next;
cur->next = NULL;
prev->next = head;
return next;           /* 无需 dummy */
```

与第 19 题不同，那道题的 `dummy` 用于统一"删除头节点"的边界情况，是必要的。
而本题中旋转操作不会出现头节点被删除的边界问题，`dummy` 纯属多余开销。

### (c) prev 命名不当

变量 `prev` 在遍历阶段结束后实际上指向链表的**尾节点**（最后一个节点），
而非某个节点的"前驱"。命名为 `prev` 容易让人误以为它是分割点的前一个节点，
实际上它的作用是记录尾节点以便在重连时将尾部接回原头部。

更清晰的命名应为 `tail`：

```c
struct ListNode *cur, *next, *tail;
/* ... */
tail = &dummy;
/* ... */
while (cur) {
    size++;
    tail = cur;
    cur = cur->next;
}
/* ... */
tail->next = head;   /* 语义更清晰：尾节点接回头部 */
```

### (d) 变量 next 的声明冗余

`next` 在函数开头声明，但只在最后四行中使用。它的作用与 `dummy.next` 完全重叠。
如果去掉 `dummy`，`next` 可以改为局部声明，或者直接用一个临时变量替代，
减少函数顶部的变量声明量。

## 复杂度分析

| 维度 | 复杂度 | 说明 |
|------|--------|------|
| **时间** | O(n) | 第一次遍历计数需要 n 步，第二次定位分割点最多 n-1 步，总计不超过 2n 步 |
| **空间** | O(1) | 仅使用 cur、next、prev、dummy 等常数个指针变量和整数变量，无额外数据结构 |

其中 `n` 为链表节点总数。

值得注意的是，本题也可以用"成环再断开"的方式实现：先将链表首尾相连形成环，
然后从适当位置断开。该方法同样是 O(n) 时间、O(1) 空间，但只需一个指针遍历到
尾节点，逻辑上更简洁。

## 总结

旋转链表的核心操作可以归纳为三步：计数求长、取模求有效旋转量、定位分割点后重连。
算法先遍历链表得到长度 `size` 和尾节点，再用 `k % size` 消除无效的完整轮旋转，
最后在第 `size - n` 个节点之前切断链表，将后半段接到前半段前面。当前实现中
`dummy` 节点是多余的，可以直接返回 `next`；`prev` 应命名为 `tail` 以准确反映
其"尾节点"的语义。整体算法 O(n) 时间、O(1) 空间，两趟扫描完成，是本题的
标准解法。
