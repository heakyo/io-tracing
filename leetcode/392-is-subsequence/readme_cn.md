## 题目描述

给定两个字符串 `s` 和 `t`，判断 `s` 是否为 `t` 的子序列。

所谓子序列，是指从 `t` 中删除一些（也可以不删除）字符，而不改变其余字符的相对顺序，所得到的新字符串。例如 `"ace"` 是 `"abcde"` 的子序列，而 `"aec"` 不是。

| 输入 | 输出 | 说明 |
|------|------|------|
| s = `"abc"`, t = `"ahbgdc"` | `true` | 依次在 t 中找到 `'a'`、`'b'`、`'c'`，且保持相对顺序 |
| s = `"axc"`, t = `"ahbgdc"` | `false` | 字符 `'x'` 在 t 中不存在，无法完成匹配 |

---

## 核心思路

算法使用**双指针**技巧，思路极其简洁：用指针 `p1` 遍历 `s`，用指针 `p2` 遍历 `t`。

对于 `s` 中的每一个字符 `*p1`，我们在 `t` 中从 `p2` 当前位置开始向前扫描，寻找与之匹配的字符。一旦找到，`p2` 停在匹配位置的下一个字符（因为后置自增），`p1` 也向前推进一步，继续匹配 `s` 的下一个字符。

如果 `p2` 一直扫描到 `t` 的末尾 `'\0'` 都没有找到匹配，说明 `t` 的剩余部分已经不包含 `s` 当前需要的字符，直接返回 `false`。

若 `s` 中所有字符都成功匹配，返回 `true`。

关键洞察：**`p2` 从不重置**。它在处理 `s` 的各个字符之间始终保持自己在 `t` 中的位置，绝不回头。这恰好保证了子序列所要求的"相对顺序"约束——前面匹配过的字符一定出现在后面匹配的字符之前。

```c
bool isSubsequence(char* s, char* t)
{
	char *p1 = s, *p2 = t;
	bool found;

	while (*p1) {
		found = false;
		while (*p2) {
			if (*p2++ == *p1) {
				found = true;
				break;
			}
		}

		if (!found)
			return false;

		p1++;
	}

	return true;
}
```

## 逐步演练

### 演练一: s = "abc", t = "ahbgdc" -> true

**外层第 1 轮: `*p1 = 'a'`**

| 内层步骤 | `*p2`（比较前） | 比较 | p2 移动后指向 | 结果 |
|-----------|-----------------|------|---------------|------|
| 1 | `'a'` | `'a' == 'a'` | `t[1] = 'h'` | `found = true`，跳出内层循环 |

匹配成功。`p1` 前进，指向 `'b'`。此时 `p2` 指向 `'h'`。

**外层第 2 轮: `*p1 = 'b'`**

| 内层步骤 | `*p2`（比较前） | 比较 | p2 移动后指向 | 结果 |
|-----------|-----------------|------|---------------|------|
| 1 | `'h'` | `'h' != 'b'` | `t[2] = 'b'` | 继续 |
| 2 | `'b'` | `'b' == 'b'` | `t[3] = 'g'` | `found = true`，跳出内层循环 |

匹配成功。`p1` 前进，指向 `'c'`。此时 `p2` 指向 `'g'`。

**外层第 3 轮: `*p1 = 'c'`**

| 内层步骤 | `*p2`（比较前） | 比较 | p2 移动后指向 | 结果 |
|-----------|-----------------|------|---------------|------|
| 1 | `'g'` | `'g' != 'c'` | `t[4] = 'd'` | 继续 |
| 2 | `'d'` | `'d' != 'c'` | `t[5] = 'c'` | 继续 |
| 3 | `'c'` | `'c' == 'c'` | `t[6] = '\0'` | `found = true`，跳出内层循环 |

匹配成功。`p1` 前进，指向 `'\0'`。外层 `while(*p1)` 条件为假，循环结束。

**返回 `true`。**

### 演练二: s = "axc", t = "ahbgdc" -> false

**外层第 1 轮: `*p1 = 'a'`**

| 内层步骤 | `*p2`（比较前） | 比较 | p2 移动后指向 | 结果 |
|-----------|-----------------|------|---------------|------|
| 1 | `'a'` | `'a' == 'a'` | `t[1] = 'h'` | `found = true`，跳出内层循环 |

匹配成功。`p1` 前进，指向 `'x'`。此时 `p2` 指向 `'h'`。

**外层第 2 轮: `*p1 = 'x'`**

| 内层步骤 | `*p2`（比较前） | 比较 | p2 移动后指向 | 结果 |
|-----------|-----------------|------|---------------|------|
| 1 | `'h'` | `'h' != 'x'` | `t[2] = 'b'` | 继续 |
| 2 | `'b'` | `'b' != 'x'` | `t[3] = 'g'` | 继续 |
| 3 | `'g'` | `'g' != 'x'` | `t[4] = 'd'` | 继续 |
| 4 | `'d'` | `'d' != 'x'` | `t[5] = 'c'` | 继续 |
| 5 | `'c'` | `'c' != 'x'` | `t[6] = '\0'` | 继续 |

`*p2` 为 `'\0'`，内层 `while(*p2)` 条件为假，循环自然结束。`found` 仍为 `false`。

检查 `!found` 为真，**返回 `false`。**

字符 `'x'` 在 `t` 的剩余部分中根本不存在，匹配失败。

## 容易踩的坑

### (a) 后置自增 `*p2++` 的微妙之处

表达式 `*p2++` 先读取 `*p2` 的值用于比较，然后无论比较结果如何都将 `p2` 向前移动一步。这意味着：即使当前字符匹配成功并 `break` 跳出内层循环，`p2` 也已经指向了匹配位置的**下一个**字符。

这个行为恰好是我们想要的。如果 `p2` 停在匹配字符本身，那么下一轮外层循环在内层中会再次看到同一个字符，导致一个 `t` 中的字符被重复使用，破坏子序列的定义。

### (b) `found` 标志的必要性

内层 `while(*p2)` 循环退出时有两种情况：

1. **找到匹配**：执行了 `break` 跳出，此时 `found == true`。
2. **未找到匹配**：`p2` 走到了 `'\0'`，循环条件为假自然退出，此时 `found == false`。

退出内层循环后，仅凭 `p2` 的值无法轻易区分这两种情况（虽然理论上可以回查，但代码会变得丑陋）。`found` 标志干净利落地解决了这个问题。

### (c) p2 从不重置——算法正确性的根基

在整个外层循环中，`p2` 始终单调向前推进，绝不回退到 `t` 的起始位置或之前的位置。这不是疏忽，而是算法正确性的核心保证。

子序列要求匹配的字符在 `t` 中保持原始的相对顺序。如果为了匹配 `s` 的下一个字符而将 `p2` 重置回开头，就可能出现后一个字符在 `t` 中的位置反而在前一个字符之前的情况，这违反了子序列的定义。

正因为 `p2` 只进不退，每次匹配都发生在上一次匹配位置之后，顺序约束自然得到满足。

## 复杂度分析

**时间复杂度: O(|s| + |t|)**

指针 `p1` 最多遍历 `s` 一遍（`|s|` 步），指针 `p2` 最多遍历 `t` 一遍（`|t|` 步）。两者都只向前移动、从不回退，因此总的字符比较次数不超过 `|s| + |t|`。

最坏情况出现在 `s` 的最后一个字符无法匹配时：`p2` 扫描完整个 `t`，同时 `p1` 可能也走了大部分 `s`。

**空间复杂度: O(1)**

只使用了两个指针 `p1`、`p2` 和一个布尔变量 `found`，占用常数级额外空间，与输入规模无关。

---

## 总结

本算法利用双指针的单向扫描特性，优雅地解决了子序列判定问题。指针 `p1` 依次提出"我需要找到这个字符"的请求，指针 `p2` 在 `t` 中从当前位置开始向前搜索作为回应。由于 `p2` 从不回退，每一次成功匹配都天然地保证了在 `t` 中的位置严格递增，从而满足子序列对相对顺序的要求。后置自增 `*p2++` 确保匹配后搜索从下一个位置继续，`found` 标志则清晰地区分了内层循环的两种退出原因。整个算法以 O(|s| + |t|) 的时间和 O(1) 的空间完成判定，简洁而高效。
