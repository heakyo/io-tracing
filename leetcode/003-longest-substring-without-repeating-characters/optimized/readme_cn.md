## 为什么要优化？

原始解法是正确的，均摊时间复杂度为 O(n)，但存在不必要的开销：

1. **每次窗口起点移动都要 memset。** 每当 `p1` 前进，原始代码重置整个 256 项 `struct mark` 数组（2048 字节）。这个常数因子会累积。
2. **嵌套循环结构。** 原始代码有外层 `while(*p1)` 和内层 `while(*p2)`，掩盖了算法的单遍扫描本质。
3. **双字段结构体。** 每个条目存储 `seen`（布尔值）和 `len`（偏移量）。优化版每个字符只需一个 int。

优化版用一个 `for` 循环遍历字符串，配合 `last[256]` 数组存储每个字符最近出现的索引。开头一次 `memset`，循环内零次。

---

## 核心思路

滑动窗口 + "最后出现位置"查找表。

维护一个窗口 `[left, right]`，始终包含不重复的字符。当 `right` 逐个前进时：

- 查找 `last[ch]`——字符 `ch` 最近一次出现的索引。
- 如果 `last[ch] >= left`，说明该字符在当前窗口内。将 `left` 移动到 `last[ch] + 1` 来收缩窗口。
- 记录 `last[ch] = right`。
- 如果当前窗口是目前最长的，更新 `max_len`。

关键洞察：永远不需要回头扫描或重置查找表。`>= left` 检查自动过滤掉过时的条目。

## 容易踩的坑

1. **用 `memset(last, 0xff, ...)` 初始化为 -1。** 对于 32 位 `int`，`0xFFFFFFFF` 在二进制补码中是 -1。这是常见的 C 语言技巧，但依赖于二进制补码表示（C23 标准保证，且所有现代硬件均如此）。用循环 `last[i] = -1` 是更"安全"的替代方案。

2. **`last[ch] >= left` 过滤过时条目。** 某个字符的 `last[ch]` 可能设置于之前的窗口位置，现在已经在 `left` 之后。`>= left` 检查自动忽略它——不需要清除旧条目。这消除了原始版本所需的逐窗口 memset。

3. **`unsigned char ch` 避免负索引。** 如果 `char` 是有符号的，值为 128-255 的字符会产生负数组索引——未定义行为！转换为 `unsigned char` 确保索引在 0-255 范围内。原始代码没有做这个处理（直接使用 `*p2`），对于扩展 ASCII 是有问题的。

4. **不需要特殊处理字符串末尾。** 原始代码的内层循环使用 `while (*p2)`，当 `p2` 到达 `'\0'` 后外层循环会继续多余的迭代直到 `*p1 == '\0'`。用 `len` 作为上界的单个 `for` 循环自然处理了空字符串（len=0，循环体不执行，返回 0）和字符串末尾，不会浪费迭代。

## 复杂度分析

| 指标 | 值 | 原因 |
|------|-----|------|
| **时间** | O(n) | 单遍扫描——`right` 恰好访问每个字符一次。`left` 只向前移动。 |
| **空间** | O(1) | 固定的 `last[256]` 数组（1 KB），与输入大小无关。 |

---

## 与原始解法对比

| | 原始版本 | 优化版本 |
|--|---------|---------|
| **算法** | 双指针 + 标记数组 | 滑动窗口 + 最后出现位置数组 |
| **时间** | O(n) 均摊 | O(n) 严格单遍 |
| **空间** | O(1)（256 个结构体 = 2 KB） | O(1)（256 个 int = 1 KB） |
| **memset 调用** | 每个窗口起点一次 | 总共一次（初始化时） |
| **循环结构** | 嵌套 `while`/`while` | 单个 `for` 循环 |
| **有符号 char 安全** | 否（`*p2` 直接用作索引） | 是（`unsigned char` 转换） |
| **代码行数** | 23（函数体） | 16（函数体） |

优化版更简洁，常数开销更低（无逐位置 memset），查找表内存减半，且安全处理扩展 ASCII。

---

## 总结

优化版使用经典的滑动窗口配合"最后出现位置"数组。一个 `for` 循环让 `right` 遍历字符串。当 `right` 处的字符上次出现在 `left` 或之后时，将 `left` 移动到该位置之后来收缩窗口。`last[256]` 数组初始化一次为 -1，之后永不重置——过时条目通过 `>= left` 检查自动过滤。这消除了原始版本的逐窗口 `memset`，将结构体简化为普通 int 数组，并将嵌套循环折叠为单遍扫描。结果是 O(n) 时间、更低的常数开销和更清晰的代码。
