## 题目描述

**LeetCode 3: Longest Substring Without Repeating Characters**

给定一个字符串 `s`，找出其中不含有重复字符的最长子串的长度。

**示例：**

| 输入 | 输出 | 说明 |
|------|------|------|
| `"abcabcbb"` | `3` | 最长无重复子串为 `"abc"` |
| `"bbbbb"` | `1` | 最长无重复子串为 `"b"` |
| `"pwwkew"` | `3` | 最长无重复子串为 `"wke"` |
| `"dvdf"` | `3` | 最长无重复子串为 `"vdf"` |

---

## 核心思路

想象你拿着一把尺子量绳子。尺子的左端是 `p1`，右端是 `p2`。你不断地把右端往右伸，直到碰到一个之前量过的重复字符。这时候你不是把左端只往右挪一格（那太笨了），而是直接跳到"重复字符第一次出现位置的下一格"——因为你已经记住了它出现在哪。

算法使用两个指针（`p1`、`p2`）和一个 256 项的标记数组 `m`，跟踪每个字符的**是否出现过**（`seen`）和**相对偏移量**（`len`，即该字符相对于 `p1` 的距离）。

具体过程：

1. 对每个窗口起点 `p1`，先用 `memset` 把整个 256 项标记数组清零。
2. 用 `p2` 从 `p1` 开始向右扫描。每遇到一个新字符，记录其 `seen = 1` 和 `len = p2 - p1`。
3. 当 `p2` 遇到一个 `seen` 为真的字符——即重复字符时，`break` 跳出内层循环。
4. 内层循环结束后（无论是遇到重复还是到达字符串末尾），统一检查当前窗口长度 `p2 - p1` 是否大于 `max`，如果是则更新。
5. 利用存储的偏移量，令 `p1 += m[*p2].len + 1`，将 `p1` 直接跳过重复字符的首次出现位置。

**关键洞察**：偏移量跳转避免了每次从 `p1 + 1` 重新开始扫描，使算法远快于朴素的 O(n²)。

## 容易踩的坑

### (a) memset 每轮都重置整个 256 项数组

```c
while(*p1) {
    memset(m, 0x0, sizeof(m));  // <-- 每次外层循环都清零！
```

每当 `p1` 移动到新位置，**整个标记数组被清零**。这意味着偏移量（`len`）是相对于当前 `p1` 的，不是绝对索引。这在概念上很重要：每一轮都是一个"全新的窗口"，之前的记录全部作废。

> 代价是每轮花费 O(256) = O(1) 时间做 memset，但换来的好处是逻辑简洁，不用担心脏数据。

### (b) 偏移量跳转是性能的关键

```c
p1 += m[*p2].len + 1;
```

这是整个算法最精妙的一行。当在位置 `p2` 发现重复字符时，`m[*p2].len` 记录的是该字符在当前窗口内**第一次出现时相对于 `p1` 的偏移量**。加 1 是为了跳到该重复字符首次出现位置的**下一个字符**。

举例：如果 `p1=0`，窗口为 `"abcXdef"`，在 `p2=7` 处再次遇到 `'X'`，而 `m['X'].len = 3`，那么 `p1 += 3 + 1 = 4`，直接跳到 `'X'` 后面的 `'d'`。

如果没有这个跳转，`p1` 只能每次前进一步（`p1++`），就变成了朴素的 O(n²) 算法。

### (c) 统一的 max 更新

```c
if (p2 - p1 > max)
    max = p2 - p1;
```

当前代码在**一个地方**更新 `max`——内层循环结束之后，无论是因为遇到重复字符还是到达字符串末尾。这比旧版本更简洁：旧版本在重复检查和末尾检查两个地方分别更新 `max`。单一更新点意味着更少的出错机会。

### (d) 不提前退出外层循环

当 `p2` 到达 `'\0'`（字符串末尾）时，代码不会 `break` 退出外层循环。它正常执行 `p1 += m[*p2].len + 1`。由于 `'\0'` 从未被标记，`m[0].len` 为 0（来自 memset），所以 `p1` 前进 1 步。外层循环继续直到 `*p1 == '\0'`。这用几次额外的（无害的）迭代换取了更简洁的代码——不需要针对字符串末尾的特殊分支。

## 复杂度分析

### 时间复杂度：O(n) 均摊

- **外层循环**：`p1` 只向前移动，永远不会后退。每次跳转距离 ≥ 1。
- **内层循环**：对于每个 `p1`，`p2` 从 `p1` 开始向右扫描。关键在于，**当发现重复字符时，`p1` 跳过的部分不会被下一轮的 `p2` 重复扫描**——因为 `p1` 跳到了重复字符首次出现位置之后，`p2` 从新的 `p1` 开始。
- 虽然每轮有一次 `memset`（O(256) = O(1)），但这不影响整体 O(n) 的性质。
- 最坏情况下（如 `"abcabc..."`），`p1` 每次前进 1 步，总共 n 步；每步内层扫描平均为常数。
- **总体**：每个字符最多被 `p2` 扫描一次后 `p1` 就跳过了它，因此均摊 O(n)。

### 空间复杂度：O(1)

- 标记数组 `m` 固定 256 项（覆盖所有 ASCII 字符），与输入字符串长度无关。
- 两个指针 `p1`、`p2` 和一个整数 `max`，均为常数空间。
- **总计**：`256 × sizeof(struct mark)` = `256 × 8` = `2048` 字节，固定不变。

## 总结

这个算法的本质是一种**带偏移量记忆的滑动窗口**。它维护一个由 `p1` 和 `p2` 界定的窗口，用一个 256 项的标记数组记录每个字符是否出现过（`seen`）以及它在窗口内的相对偏移（`len`）。当 `p2` 遇到重复字符时，内层循环 `break` 退出，然后在一个统一的位置检查并更新 `max`。接着算法利用预先记录的偏移量将 `p1` 直接跳到重复字符首次出现位置的下一格，一步到位地跳过所有不可能产生更长子串的起点。整个算法在均摊 O(n) 时间、O(1) 空间内完成——既简洁又高效。
