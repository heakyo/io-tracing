## 题目描述

**LeetCode 3: Longest Substring Without Repeating Characters**

给定一个字符串 `s`，找出其中不含有重复字符的最长子串的长度。

**示例：**

| 输入 | 输出 | 说明 |
|------|------|------|
| `"abcabcbb"` | `3` | 最长无重复子串为 `"abc"` |
| `"bbbbb"` | `1` | 最长无重复子串为 `"b"` |
| `"pwwkew"` | `3` | 最长无重复子串为 `"wke"` |
| `"dvdf"` | `3` | 最长无重复子串为 `"vdf"` |

---

## 核心思路

想象你拿着一把尺子量绳子。尺子的左端是 `p1`，右端是 `p2`。你不断地把右端往右伸，直到碰到一个之前量过的重复字符。这时候你不是把左端只往右挪一格（那太笨了），而是直接跳到"重复字符第一次出现位置的下一格"——因为你已经记住了它出现在哪。

算法使用两个指针（`p1`、`p2`）和一个 256 项的标记数组 `m`，跟踪每个字符的**出现次数**（`cnt`）和**相对偏移量**（`offset`，即该字符相对于 `p1` 的距离）。

具体过程：

1. 对每个窗口起点 `p1`，先用 `memset` 把整个 256 项标记数组清零。
2. 用 `p2` 从 `p1` 开始向右扫描。每遇到一个新字符，记录其 `cnt = 1` 和 `offset = p2 - p1`。
3. 当 `p2` 遇到一个 `cnt > 0` 的字符——即重复字符时：
   - 计算当前窗口长度 `p2 - p1`，更新 `max`。
   - 利用存储的偏移量，令 `p1 += m[*p2].offset + 1`，将 `p1` 直接跳过重复字符的首次出现位置。
4. 如果 `p2` 扫到字符串末尾（`'\0'`）都没遇到重复，更新 `max` 并直接退出——后面不可能更长了。

**关键洞察**：偏移量跳转避免了每次从 `p1 + 1` 重新开始扫描，使算法远快于朴素的 O(n²)。

### 演练一：`"abcabcbb"`

```

索引:   0  1  2  3  4  5  6  7
```

### 演练二：`"dvdf"`

```
字符串: d  v  d  f
索引:   0  1  2  3

══════════════════════════════════════════════════════════════
第 1 轮：p1=0('d'), memset 清零标记数组
──────────────────────────────────────────────────────────────
  p2=0  'd': cnt=0 → 记录 cnt=1, offset=0        m['d']={1,0}
  p2=1  'v': cnt=0 → 记录 cnt=1, offset=1        m['v']={1,1}
  p2=2  'd': cnt=1 → 重复！
        当前窗口长度 = p2 - p1 = 2 - 0 = 2 → max = 2
        跳转：p1 += m['d'].offset + 1 = 0 + 1 = 1
        → p1 移至索引 1

══════════════════════════════════════════════════════════════
第 2 轮：p1=1('v'), memset 清零标记数组
──────────────────────────────────────────────────────────────
  p2=1  'v': cnt=0 → 记录 cnt=1, offset=0        m['v']={1,0}
  p2=2  'd': cnt=0 → 记录 cnt=1, offset=1        m['d']={1,1}
  p2=3  'f': cnt=0 → 记录 cnt=1, offset=2        m['f']={1,2}
  p2=4  '\0': 内层循环结束
        *p2 == '\0', p2 - p1 = 4 - 1 = 3 > max(2) → max = 3
        break 退出外层循环！

══════════════════════════════════════════════════════════════

返回 max = 3 ✓
```

> **注意**：在第 2 轮中，`p2` 扫到字符串末尾都没遇到重复字符。此时剩余字符串已经全部扫完，不可能找到更长的子串，所以直接 `break` 退出。这是一个重要的优化。

## 容易踩的坑

### (a) memset 每轮都重置整个 256 项数组

```c
while(*p1) {
    memset(m, 0x0, sizeof(struct mark)*256);  // <-- 每次外层循环都清零！
```

每当 `p1` 移动到新位置，**整个标记数组被清零**。这意味着偏移量（`offset`）是相对于当前 `p1` 的，不是绝对索引。这在概念上很重要：每一轮都是一个"全新的窗口"，之前的记录全部作废。

> 代价是每轮花费 O(256) = O(1) 时间做 memset，但换来的好处是逻辑简洁，不用担心脏数据。

### (b) 偏移量跳转是性能的关键

```c
p1 += m[*p2].offset + 1;
```

这是整个算法最精妙的一行。当在位置 `p2` 发现重复字符时，`m[*p2].offset` 记录的是该字符在当前窗口内**第一次出现时相对于 `p1` 的偏移量**。加 1 是为了跳到该重复字符首次出现位置的**下一个字符**。

举例：如果 `p1=0`，窗口为 `"abcXdef"`，在 `p2=7` 处再次遇到 `'X'`，而 `m['X'].offset = 3`，那么 `p1 += 3 + 1 = 4`，直接跳到 `'X'` 后面的 `'d'`。

如果没有这个跳转，`p1` 只能每次前进一步（`p1++`），就变成了朴素的 O(n²) 算法。

### (c) 字符串末尾检查提前退出

```c
if (*p2 == '\0' && p2 - p1 > max) {
    max = p2 - p1;
    break;               // <-- break 的是外层 while(*p1)
}
```

当 `p2` 扫到字符串末尾（`'\0'`）都没遇到重复字符时，说明从 `p1` 到字符串结尾都是不重复的。此时：

- 从 `p1` 到末尾的长度一定 ≥ 从 `p1+k` 到末尾的长度（对任何 `k > 0`）
- 因此后续不可能出现更长的无重复子串
- 直接 `break` 退出外层循环，避免无谓的计算

这个优化在输入字符串末尾有一段很长的不重复子串时特别有效。

### (d) break 前的 `m[*p2].cnt = 0` 重置

```c
if (m[*p2].cnt > 0) {
    m[*p2].cnt = 0;       // <-- 重置 cnt，但 offset 不动
    if (p2 - p1 > max)
        max = p2 - p1;
    break;
}
```

这里将 `cnt` 重置为 0，但注意 `offset` 字段**没有被重置**——因为紧接着 `break` 之后代码要用 `m[*p2].offset` 来计算 `p1` 的跳转距离。`cnt` 的重置是防御性编程（break 后紧跟 memset，所以理论上不重置也行），但 `offset` 的保留是**必需的**，它在 break 之后的第 44 行被读取使用。

## 复杂度分析

### 时间复杂度：O(n) 均摊

- **外层循环**：`p1` 只向前移动，永远不会后退。每次跳转距离 ≥ 1。
- **内层循环**：对于每个 `p1`，`p2` 从 `p1` 开始向右扫描。关键在于，**当发现重复字符时，`p1` 跳过的部分不会被下一轮的 `p2` 重复扫描**——因为 `p1` 跳到了重复字符首次出现位置之后，`p2` 从新的 `p1` 开始。
- 虽然每轮有一次 `memset`（O(256) = O(1)），但这不影响整体 O(n) 的性质。
- 最坏情况下（如 `"abcabc..."`），`p1` 每次前进 1 步，总共 n 步；每步内层扫描平均为常数。
- **总体**：每个字符最多被 `p2` 扫描一次后 `p1` 就跳过了它，因此均摊 O(n)。

### 空间复杂度：O(1)

- 标记数组 `m` 固定 256 项（覆盖所有 ASCII 字符），与输入字符串长度无关。
- 两个指针 `p1`、`p2` 和一个整数 `max`，均为常数空间。
- **总计**：`256 × sizeof(struct mark)` = `256 × 8` = `2048` 字节，固定不变。

## 总结

这个算法的本质是一种**带偏移量记忆的滑动窗口**。它维护一个由 `p1` 和 `p2` 界定的窗口，用一个 256 项的标记数组记录每个字符是否出现过以及它在窗口内的相对偏移。当 `p2` 遇到重复字符时，算法不是笨拙地把 `p1` 挪一步，而是利用预先记录的偏移量将 `p1` 直接跳到重复字符首次出现位置的下一格，一步到位地跳过所有不可能产生更长子串的起点。配合字符串末尾的提前退出优化，整个算法在均摊 O(n) 时间、O(1) 空间内完成——既简洁又高效。
