## 问题描述

给定一个整数数组 `nums`，找出所有满足以下条件的三元组 `[nums[i], nums[j], nums[k]]`：

- 三个下标互不相同：`i != j`、`i != k`、`j != k`
- 三数之和为零：`nums[i] + nums[j] + nums[k] == 0`

返回的结果中**不能包含重复的三元组**。

打个比方：你手里有一把标着不同数字的卡牌，你要从中选出三张，使它们上面的数字加起来恰好等于零。而且，即使不同位置的卡牌上写着相同的数字，只要最终选出来的三个数字组合相同，就只算一种。

---

## 核心思路

如果用最笨的方法——三重循环枚举所有组合——时间复杂度是 O(n³)，太慢了。我们需要一个更聪明的办法。

核心想法只有一句话：**固定一个元素当"锚点"，剩下两个用双指针从两头往中间逼近**。

具体步骤如下：

1. **对数组排序**。排序是一切的基础：它让我们能用双指针，也让去重变得简单。

2. **遍历每个元素 `nums[i]`（`i` 从 `0` 到 `n-3`）**，把它当作三元组的锚点。对于每个锚点：

	- **若 `nums[i] > 0`，直接 `break`**。为什么？数组已排序，`nums[i]` 后面的元素只会更大。三个正数之和不可能为零，所以后续所有的 `i` 都不需要再看了。

	- **若 `i > 0` 且 `nums[i] == nums[i-1]`，则 `continue`**。这是锚点层面的去重——同一个值做锚点只需要做一次，否则会产生重复三元组。

	- **设双指针 `left = i + 1`，`right = n - 1`**。

	- **当 `left < right` 时**，计算三数之和：

		```
		sum = nums[i] + nums[left] + nums[right]
		```

		- **若 `sum == 0`**：找到一个有效三元组！记录下来。然后跳过左侧和右侧的所有重复值，最后双指针同时向内移动（`left++`，`right--`）。
		- **若 `sum < 0`**：和太小了，需要更大的值，所以 `left++`。
		- **若 `sum > 0`**：和太大了，需要更小的值，所以 `right--`。

想象一下：排序后的数组是一条从小到大的数轴。锚点 `nums[i]` 固定在左边某个位置，`left` 和 `right` 像两只手分别从锚点右侧的两端向中间合拢。和太小就把左手右移（取更大的值），和太大就把右手左移（取更小的值）。这样，对于每个锚点，双指针只需要扫描一趟就能找到所有匹配的组合。

此方法系统地探索所有有效三元组，不会遗漏任何一个。

## 逐步推演

以输入 `[-1, 0, 1, 2, -1, -4]` 为例。

**排序后：`[-4, -1, -1, 0, 1, 2]`**（下标 0~5）

---

### i = 0，nums[0] = -4

```
数组：  [-4, -1, -1,  0,  1,  2]
下标：    0    1    2   3   4   5
          i   left                right
```

- `left = 1`，`right = 5`
- `sum = -4 + (-1) + 2 = -3 < 0` → `left++`
- `left = 2`，`right = 5`
- `sum = -4 + (-1) + 2 = -3 < 0` → `left++`
- `left = 3`，`right = 5`
- `sum = -4 + 0 + 2 = -2 < 0` → `left++`
- `left = 4`，`right = 5`
- `sum = -4 + 1 + 2 = -1 < 0` → `left++`
- `left = 5`，`left >= right`，内层循环结束。

**结论：nums[0] = -4 作为锚点时，无法找到和为 0 的三元组。**

---

### i = 1，nums[1] = -1

```
数组：  [-4, -1, -1,  0,  1,  2]
下标：    0    1    2   3   4   5
               i   left            right
```

- `left = 2`，`right = 5`
- `sum = -1 + (-1) + 2 = 0` ✅ → 记录 **[-1, -1, 2]**
	- 跳过重复：`nums[left] == nums[left+1]`？`nums[2]=-1`，`nums[3]=0`，不相等，跳过去重。
	- 跳过重复：`nums[right] == nums[right-1]`？`nums[5]=2`，`nums[4]=1`，不相等，跳过去重。
	- `left++` → `left = 3`，`right--` → `right = 4`

```
数组：  [-4, -1, -1,  0,  1,  2]
下标：    0    1    2   3   4   5
               i       left right
```

- `left = 3`，`right = 4`
- `sum = -1 + 0 + 1 = 0` ✅ → 记录 **[-1, 0, 1]**
	- 跳过重复后，`left++` → `left = 4`，`right--` → `right = 3`
	- `left >= right`，内层循环结束。

---

### i = 2，nums[2] = -1

`nums[2] == nums[1]`（都是 -1），**跳过**，避免产生重复三元组。

---

### i = 3，nums[3] = 0

```
数组：  [-4, -1, -1,  0,  1,  2]
下标：    0    1    2   3   4   5
                        i  left right
```

- `left = 4`，`right = 5`
- `sum = 0 + 1 + 2 = 3 > 0` → `right--`
- `right = 4`，`left >= right`，内层循环结束。

---

### 最终结果

```
[[-1, -1, 2], [-1, 0, 1]]
```

## 易错点分析

### (a) 三层去重——缺一不可

去重不是在一个地方做就够了，而是需要在**三个位置**同时去重：

1. **锚点去重**：`if (i > 0 && nums[i] == nums[i - 1]) continue;`
	跳过与前一个锚点值相同的元素，避免以相同的值为起点重复搜索。

2. **左指针去重**：`while (left < right && nums[left] == nums[left + 1]) left++;`
	找到有效三元组后，跳过左指针右侧所有相同的值。

3. **右指针去重**：`while (left < right && nums[right] == nums[right - 1]) right--;`
	找到有效三元组后，跳过右指针左侧所有相同的值。

打个比方：这就像安检有三道门——外门检查锚点、中门检查左指针、里门检查右指针。少了任何一道门，重复的三元组就会"溜"进结果里。

---

### (b) 提前终止优化

```c
if (nums[i] > 0)
	break;
```

数组已经排好序了。如果当前锚点 `nums[i]` 已经大于零，那么 `nums[left]` 和 `nums[right]` 只会更大（或相等）。三个非负数之和不可能为零（除非全是零，但那时 `nums[i] == 0`，不会触发这个条件）。所以后面所有的 `i` 都可以直接跳过。

注意是 `break` 而不是 `continue`——我们是要终止整个外层循环，而不仅仅跳过当前迭代。

---

### (c) 找到三元组后必须同时移动双指针

```c
left++;
right--;
```

找到一个 `sum == 0` 的三元组后，如果只移动 `left` 或只移动 `right`，会怎样？由于 `sum` 之前恰好为零，只动一个指针后新的 `sum` 一定不为零，但更关键的是：之后可能再次回到同一个组合，或者遗漏其他有效组合。正确做法是同时移动两个指针，进入全新的搜索区间。

---

### (d) 去重循环的执行顺序

去重循环在指针最终递增**之前**执行：

```c
// 先跳过重复值
while (left < right && nums[left] == nums[left + 1])
	left++;
while (left < right && nums[right] == nums[right - 1])
	right--;
// 再移动到新值上
left++;
right--;
```

这个顺序很重要。去重循环让 `left` 停在最后一个重复值上，然后 `left++` 使它落在下一个**不同的**值上。如果反过来——先 `left++` 再去重——就会多走一步，跳过一个本该检查的值。

## 复杂度分析

### 时间复杂度：O(n²)

- **排序**：O(n log n)
- **外层循环**：遍历每个元素作为锚点，共 O(n) 次。
- **内层双指针**：对于每个锚点，`left` 和 `right` 各自单调移动，最多扫描 O(n) 个元素。
- **总计**：O(n log n) + O(n) × O(n) = **O(n²)**。排序的 O(n log n) 被 O(n²) 主导。

### 空间复杂度：O(1) 额外空间

- 算法本身只使用了常数个变量（`i`、`left`、`right`、`sum`）。
- 输出数组的空间不计入额外空间复杂度（那是题目要求返回的结果）。
- 排序使用的栈空间为 O(log n)，通常也不计入。
- 因此额外空间复杂度为 **O(1)**。

---

## 总结

标准的**排序 + 锚点 + 双指针**方法是解决三数之和问题的最优方案。

它的精妙之处在于：通过**固定一个元素**，把三数之和问题降维成了两数之和问题；再利用**数组有序**这个性质，用双指针一次扫描就能找到所有匹配的二元组。这就像一个经验老到的侦探，先锁定一个嫌疑人，然后在剩下的人里用排除法快速找到同伙。

**三层去重**（锚点去重、左指针去重、右指针去重）确保结果中没有重复的三元组。**提前终止**（`nums[i] > 0` 时 `break`）进一步减少无效搜索。

- **时间复杂度**：O(n²)
- **额外空间复杂度**：O(1)

这种方法系统地探索了每一种有效组合，既不会遗漏，也不会重复——这正是算法之美。
