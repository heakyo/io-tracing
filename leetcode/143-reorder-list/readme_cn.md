## 题目描述

给你一个单链表，节点依次为 L0 -> L1 -> L2 -> ... -> Ln-1 -> Ln。

要求你把它重新排列成：L0 -> Ln -> L1 -> Ln-1 -> L2 -> Ln-2 -> ...

通俗来说，就像一副扑克牌，你同时从头和尾各取一张，交替放在一起。第一张从头取，第二张从尾取，第三张从头取，第四张从尾取......直到所有牌都放完。

约束条件：

- 必须原地操作，不能创建新链表再拷贝回去。
- 不能修改节点的值，只能改变节点的 `next` 指针。

---

## 核心思路

算法采用"三步法"，把一个看似复杂的重排问题拆解为三个简单的子问题：

### 第一步：分割 -- 把链表一切两半

先遍历整个链表计算总节点数 `size`，然后计算需要反转的后半部分长度
`revert_size = size >> 1`（即 size / 2 向下取整）。前半部分保留
`size - revert_size` 个节点，在分割点处断开链表。

类比：把一条绳子从中间剪断，分成前后两段。如果绳子长度是奇数，前半段比后半段多一个结。

### 第二步：反转 -- 把后半段倒过来

对后半部分链表做原地反转。这样原来排在最末尾的节点变成了后半段的头节点，正好是我们需要最先插入的那个。

类比：把后半段绳子翻转过来，尾巴变成头。

### 第三步：合并 -- 交替穿插两个半链表

同时从两个半链表的头部出发，每次从前半段取一个节点、从后半段取一个节点，交替拼接起来，直到其中一个链表用完为止。

类比：左手拿前半段牌，右手拿后半段牌，左右手轮流出牌，拼成一叠。

## 逐步推演

以链表 `[1, 2, 3, 4, 5]` 为例（size = 5）。

### 第一步：计数并分割

```
size = 5
revert_size = 5 >> 1 = 2
前半长度 = size - revert_size = 3
```

从 head 出发走 3 步（i = 0, 1, 2），`pre` 停在第 3 个节点，`cur` 指向第 4 个节点。断开 `pre->next = NULL`：

```
前半链表: 1 -> 2 -> 3 -> NULL
后半链表: 4 -> 5 -> NULL     (cur 指向 4)
```

### 第二步：反转后半部分

对 `4 -> 5 -> NULL` 执行原地反转：

```
初始状态:  cur = 4,  revert_head = NULL

第 1 轮:
  tmp = cur->next = 5
  cur->next = revert_head = NULL    =>  4 -> NULL
  revert_head = cur = 4
  cur = tmp = 5

第 2 轮:
  tmp = cur->next = NULL
  cur->next = revert_head = 4       =>  5 -> 4 -> NULL
  revert_head = cur = 5
  cur = tmp = NULL

循环结束。
```

反转后：

```
前半链表:  1 -> 2 -> 3 -> NULL
后半链表:  5 -> 4 -> NULL          (revert_head 指向 5)
```

### 第三步：交替合并

```
cur = head = 1,  revert_head = 5

第 1 轮:
  tmp = 1, tmp2 = 5
  cur = 1->next = 2,  revert_head = 5->next = 4
  拼接: 1 -> 5 -> 2         (tmp->next = tmp2, tmp2->next = cur)

第 2 轮:
  tmp = 2, tmp2 = 4
  cur = 2->next = 3,  revert_head = 4->next = NULL
  拼接: 1 -> 5 -> 2 -> 4 -> 3   (tmp->next = tmp2, tmp2->next = cur)

第 3 轮:
  cur = 3, revert_head = NULL  =>  条件不满足，循环结束。
```

最终结果：

```
1 -> 5 -> 2 -> 4 -> 3 -> NULL
```

与预期 `[1, 5, 2, 4, 3]` 一致。

## 易错点分析

### 1. 计数分割 vs 快慢指针

本算法选择先遍历一遍计算 `size`，再根据 `size` 计算分割点。另一种常见做法是用快慢指针（快指针走两步、慢指针走一步）直接定位中点。两种方法都可行，但计数法的逻辑更直观，不容易在奇偶长度上出错。快慢指针法的优势是只需一次遍历就能找到中点，但需要格外注意边界条件。

### 2. 奇偶长度的分割点

这是最容易出错的地方。`revert_size = size >> 1` 对偶数和奇数有不同效果：

| size | revert_size | 前半长度 | 前半 | 后半 |
|------|-------------|---------|------|------|
| 4    | 2           | 2       | 1,2  | 3,4  |
| 5    | 2           | 3       | 1,2,3| 4,5  |
| 6    | 3           | 3       | 1,2,3| 4,5,6|
| 7    | 3           | 4       | 1,2,3,4 | 5,6,7 |

关键点：当 size 为奇数时，前半部分比后半部分多一个节点。这是正确的，因为合并时多出来的那个节点自然留在原位，不需要额外处理。如果反过来让后半段更长，合并完成后会有一个悬空节点需要单独处理。

### 3. 合并终止条件

合并循环的条件是 `cur && revert_head`，即两个链表都不为空。因为前半段长度大于等于后半段，所以后半段一定先用完（或同时用完）。循环结束时：

- 偶数长度：两个链表同时用完。
- 奇数长度：`revert_head` 先变为 NULL，`cur` 还剩最后一个节点，而这个节点已经在上一轮通过 `tmp2->next = cur` 正确连接了。

如果错误地只检查其中一个条件（如只检查 `revert_head`），当前半段先用完时会导致空指针访问。

### 4. 断开链表的时机

在分割阶段，必须执行 `pre->next = NULL` 来断开前后两半。如果忘记这一步，前半链表仍然连着后半部分，合并阶段会产生环形链表，导致死循环。

## 复杂度分析

**时间复杂度: O(n)**

- 第一步计数：遍历整个链表，O(n)。
- 第一步分割：遍历前半段，O(n/2)。
- 第二步反转：遍历后半段，O(n/2)。
- 第三步合并：遍历后半段长度，O(n/2)。
- 总计：O(n) + O(n/2) + O(n/2) + O(n/2) = O(n)。

**空间复杂度: O(1)**

只使用了固定数量的指针变量（`revert_head`、`cur`、`pre`、`tmp`、`tmp2`）和整数变量（`size`、`revert_size`、`i`），不随输入规模增长，因此为常数空间。

## 总结

重排链表问题的核心在于"从两端交替取节点"，但单链表只能从前往后遍历，无法直接从尾部取节点。解决方案是将问题拆解为三个独立的子操作：先计数分割，将链表从中间断开为前后两半；再对后半段原地反转，使得"从尾部取"变成"从头部取"；最后将两个半链表交替合并。每个子操作都是链表的基本操作，组合在一起就解决了整个问题。算法只需遍历链表常数次，时间复杂度为 O(n)；只使用固定数量的指针变量，空间复杂度为 O(1)。
