## 问题描述

给定链表的头节点 `head`，判断链表中是否存在**环**。

如果链表中某个节点的 `next` 指针指向了链表中之前已经访问过的节点，则说明存在环。
换句话说，链表的尾节点没有指向 `NULL`，而是指向了链表中某个中间节点，形成了一个闭合回路。

题目中用 `pos` 表示尾节点所连接的节点索引（从 0 开始计数），`pos = -1` 表示无环。
注意 `pos` 仅用于描述链表结构，并不会作为参数传入函数。

**约束条件：**

- 链表节点数量范围：`0` 到 `10^4`
- 节点值范围：`-10^5 <= val <= 10^5`
- 返回 `true`（有环）或 `false`（无环）

## 核心思路

本题使用 **Floyd 环检测算法**（龟兔赛跑算法）。

想象一下：两个人在一条跑道上跑步，一个跑得快，一个跑得慢。
如果跑道是**直线**，快跑者会率先到达终点，两人永远不会相遇。
如果跑道是**环形**的，快跑者终将从后面追上慢跑者——他们一定会在某个时刻相遇。

把这个直觉翻译成代码：

- `sp`（慢指针，slow pointer）：每次走 **1** 步
- `qp`（快指针，quick pointer）：每次走 **2** 步

如果链表有环，快指针会在环内不断绕圈，最终追上慢指针。
如果链表无环，快指针会率先遇到 `NULL`，循环自然结束。

本实现中，`sp` 从 `head` 开始，`qp` 从 `head->next` 开始（两者相距一步），
采用**先比较再前进**的策略。

```c
bool hasCycle(struct ListNode *head)
{
	struct ListNode *qp, *sp;

	if (head == NULL || head->next == NULL)
		return false;

	sp = head;
	qp = sp->next;

	while (qp && qp->next) {
		if (qp == sp)
			return true;

		sp = sp->next;
		qp = qp->next->next;
	}

	return false;
}
```

## 逐步推演

以 `[3, 2, 0, -4]`，`pos = 1` 为例。

链表结构如下：

```
Node0(3) -> Node1(2) -> Node2(0) -> Node3(-4)
               ^                        |
               |________________________|
```

尾节点 `Node3(-4)` 的 `next` 指回 `Node1(2)`，形成环。

**初始化：**

- `head != NULL` 且 `head->next != NULL` → 不提前返回
- `sp = head` → `Node0(3)`
- `qp = head->next` → `Node1(2)`

**迭代过程：**

| 轮次 | 检查 `qp == sp` | 结果 | `sp` 移动后 | `qp` 移动后 |
|:----:|:---------------:|:----:|:-----------:|:-----------:|
| 1 | `Node1 == Node0` ? | 不等 | `Node0→Node1(2)` | `Node1→next→next = Node2→next = Node3(-4)` |
| 2 | `Node3 == Node1` ? | 不等 | `Node1→Node2(0)` | `Node3→next→next = Node1→next = Node2(0)` |
| 3 | `Node2 == Node2` ? | **相等！** | — | — |

第 3 轮，`qp` 和 `sp` 都指向 `Node2(0)`，发现相遇，返回 `true`。

**为什么它们一定会相遇？** 当两个指针都进入环后，快指针每一步比慢指针多走一个节点，
也就是说两者之间的距离每一步缩小 1。无论初始差距多大，最多经过环长度次迭代后，
距离就会缩小到 0——即相遇。

## 易错点分析

### (a) 起始位置：先比较 vs 先前进

本实现让 `sp` 从 `head` 出发、`qp` 从 `head->next` 出发，两者**初始相距一步**，
然后在循环中**先比较再前进**。

更常见的标准写法是两者**都从 `head` 出发**，循环中**先前进再比较**：

```c
sp = head;
qp = head;
while (qp && qp->next) {
	sp = sp->next;
	qp = qp->next->next;
	if (sp == qp)
		return true;
}
```

两种写法**逻辑等价**，都能正确检测环。但本版本较不常见，在面试或代码审查中
可能会引起额外的疑问。

### (b) `head->next == NULL` 的提前返回

```c
if (head == NULL || head->next == NULL)
	return false;
```

其中 `head->next == NULL` 在技术上是**冗余**的。因为如果只有一个节点且无环，
`qp = head->next` 为 `NULL`，`while (qp && qp->next)` 条件不满足，
循环不会执行，直接返回 `false`。

不过，提前返回可以视为一个**微优化**——避免了不必要的赋值操作。

### (c) 变量命名

`sp` 和 `qp` 分别是 slow pointer 和 quick pointer 的缩写。
虽然含义可以推断，但不如 `slow`/`fast` 或 `tortoise`/`hare` 来得直观。
好的命名应该让读者**不需要查看注释**就能理解变量的用途。

### (d) 为什么快指针一定能追上慢指针？

当两个指针都进入环后，设它们之间的距离为 `d`（沿环的方向计算）。
每一步，快指针走 2 步、慢指针走 1 步，距离缩小 1。经过 `d` 步后距离变为 0，
即相遇。因为 `d` 不超过环的长度，所以最多经过**环长度次迭代**即可检测到环。

## 复杂度分析

| 指标 | 复杂度 | 说明 |
|:----:|:------:|:----:|
| 时间 | **O(n)** | 无环时快指针遍历整个链表；有环时最多额外绕环一圈 |
| 空间 | **O(1)** | 仅使用两个指针变量，无需额外数据结构 |

其中 `n` 为链表节点总数。

与之对比，使用哈希表记录访问过的节点也能检测环，但空间复杂度为 O(n)。
Floyd 算法的优势正在于它只用**常数空间**就完成了检测。

---

## 总结

Floyd 环检测算法的核心是一个极其优雅的直觉：**两个速度不同的指针在环中必然相遇**。

本实现正确且高效——O(n) 时间、O(1) 空间——但在代码风格上有值得注意的地方：

1. 变量命名 `sp`/`qp` 不够直观，建议使用 `slow`/`fast`
2. 起始位置选择（相距一步，先比较后前进）不如标准写法（同一起点，先前进后比较）常见
3. `head->next == NULL` 的提前返回是冗余的微优化

这些都不影响正确性，但会影响代码的**可读性**和**可维护性**。
写出正确的算法只是第一步，写出让别人（包括未来的自己）能轻松读懂的代码才是目标。
