## 为什么要优化？

原始解法在算法层面已经是最优的——O(m+n) 时间，O(1) 空间。这里的"优化"主要是**代码清晰度和整洁性**：

1. **移除未使用的 `swap` 函数。** 原始代码定义了 `swap` 但从未调用——属于增加困惑的死代码。
2. **用索引访问替代指针运算。** 使用整数索引 `i`、`j`、`k` 比使用指针 `p1`、`p2`、`p` 配合 `nums1 <= p1` 这样的地址比较更容易理解。边界判断变成简单的 `i >= 0`，而不是地址比较。
3. **相同算法，更干净的实现。** 从后往前合并的逻辑完全相同。

---

## 核心思路

与原始版本相同：从后往前填充 `nums1`。三个索引——`i`（`nums1` 数据末尾）、`j`（`nums2` 末尾）、`k`（写入位置）——从尾部开始向前移动。每一步将两个末尾元素中较大的放到位置 `k`。主循环结束后，复制 `nums2` 的剩余元素。

用整数索引让逻辑一目了然：

```c
int i = m - 1;     // nums1 数据的最后一个元素
int j = n - 1;     // nums2 的最后一个元素
int k = m + n - 1; // nums1 的最后一个槽位
```

循环条件 `i >= 0 && j >= 0` 含义直白——任一数组用完就停止。

## 逐步演练

`nums1 = [3, 4, 0, 0]`（m=2），`nums2 = [1, 2]`（n=2）：

```
i=1 (值 4), j=1 (值 2), k=3

第1步: nums2[1]=2 < nums1[1]=4 -> nums1[3]=4, i=0, k=2
  nums1: [3, 4, 0, 4]

第2步: nums2[1]=2 < nums1[0]=3 -> nums1[2]=3, i=-1, k=1
  nums1: [3, 4, 3, 4]

主循环退出（i < 0）。

复制 nums2 剩余元素: nums1[1]=2, nums1[0]=1
  nums1: [1, 2, 3, 4]
```

`nums1 = [1, 2, 3, 0, 0, 0]`（m=3），`nums2 = [2, 5, 6]`（n=3）：

```
i=2 (值 3), j=2 (值 6), k=5

第1步: 6 > 3 -> nums1[5]=6, j=1, k=4
第2步: 5 > 3 -> nums1[4]=5, j=0, k=3
第3步: 2 < 3 -> nums1[3]=3, i=1, k=2
第4步: 2 = 2 -> nums1[2]=2（取 nums1 的）, i=0, k=1
第5步: 2 > 1 -> nums1[1]=2, j=-1, k=0

主循环退出（j < 0）。没有 nums2 剩余元素。
结果: [1, 2, 2, 3, 5, 6]
```

## 容易踩的坑

1. **索引 -1 是"已用完"的哨兵。** 当 `i` 降到 -1，`nums1` 的数据用完了。当 `j` 降到 -1，`nums2` 用完了。简单的 `>= 0` 检查清晰明了。原始版本的指针比较 `nums1 <= p1` 效果相同，但不够直观。

2. **只有 `nums2` 的剩余需要复制。** 如果主循环后 `i >= 0` 但 `j < 0`，`nums1` 的剩余元素已经在位置 `0..i`——正好是它们该在的位置。不需要复制。这是该算法能做到 O(1) 空间的关键洞察。

3. **相等元素取 `nums1` 的。** 当 `nums2[j] == nums1[i]` 时，else 分支取 `nums1` 的元素。这保持了稳定性（相等元素中 `nums1` 的排在前面），虽然题目并不要求稳定性。

## 复杂度分析

| 指标 | 值 | 原因 |
|------|-----|------|
| **时间** | O(m + n) | m+n 个元素各被放置恰好一次 |
| **空间** | O(1) | 三个整数变量；合并在原地进行 |

## 与原始解法对比

| | 原始版本 | 优化版本 |
|--|---------|---------|
| **算法** | 从后往前合并 | 从后往前合并（相同） |
| **时间** | O(m + n) | O(m + n) |
| **空间** | O(1) | O(1) |
| **循环边界** | 指针比较（`nums1 <= p1`） | 索引比较（`i >= 0`） |
| **死代码** | 未使用的 `swap()` 函数 | 已移除 |
| **可读性** | 指针运算需要心算地址 | 整数索引一目了然 |

两者算法完全相同。优化版是代码整理，不是复杂度改进。

---

## 总结

优化版用整数索引变量（`i`、`j`、`k`）替代指针运算，使从后往前合并的逻辑更容易理解。移除了未使用的 `swap` 函数。算法不变——从后往前比较两个数组最大的剩余元素，将较大的填入 `nums1`。复杂度保持 O(m+n) 时间和 O(1) 空间，这对于此问题已经是最优解。
