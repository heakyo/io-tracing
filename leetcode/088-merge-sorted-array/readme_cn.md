## 题目描述

给定两个按非递减顺序排列的整数数组 `nums1` 和 `nums2`，以及两个整数 `m` 和 `n`，分别表示 `nums1` 和 `nums2` 中的元素个数。

将 `nums2` 合并到 `nums1` 中，使 `nums1` 成为一个有序数组。`nums1` 的总长度为 `m + n`——前 `m` 个元素是实际数据，后 `n` 个位置用零填充（作为合并结果的占位符）。

示例：

```
输入:  nums1 = [1, 2, 3, 0, 0, 0], m = 3
       nums2 = [2, 5, 6],           n = 3
输出:  nums1 = [1, 2, 2, 3, 5, 6]
```

---

## 核心思路

想象你有两叠按分数排好序的试卷，还有一张刚好能放下所有试卷的长桌。如果从桌子**前端**开始放，你可能会覆盖第一叠中还没处理的试卷。

诀窍是：**从后面开始放**。比较两叠中最大的那张，把更大的放到桌子末尾，然后向前推进。因为 `nums1` 尾部有空位，从后往前填不会覆盖还没处理的元素。

代码使用三个指针：
- `p1` — 指向 `nums1` 中最后一个有效元素（索引 `m - 1`）
- `p2` — 指向 `nums2` 的最后一个元素（索引 `n - 1`）
- `p` — 指向 `nums1` 的最后一个槽位（索引 `m + n - 1`）

每一步比较 `*p1` 和 `*p2`，将较大的复制到 `*p`，并将对应指针向前移。主循环结束后，如果 `nums2` 还有剩余元素，就复制过来。不需要复制 `nums1` 的剩余元素——它们本来就在正确的位置上。

## 逐步演练

以 `nums1 = [4, 0, 0, 0, 0, 0]`（m=1）和 `nums2 = [1, 2, 3, 5, 6]`（n=5）为例：

```
初始状态:
  nums1: [4, 0, 0, 0, 0, 0]
  p1=0 (值 4), p2=4 (值 6), p=5

第1步: *p2=6 > *p1=4 -> nums1[5]=6, p2=3, p=4
  nums1: [4, 0, 0, 0, 0, 6]

第2步: *p2=5 > *p1=4 -> nums1[4]=5, p2=2, p=3
  nums1: [4, 0, 0, 0, 5, 6]

第3步: *p2=3 < *p1=4 -> nums1[3]=4, p1 移到 nums1 之前, p=2
  nums1: [4, 0, 0, 4, 5, 6]

主循环退出（p1 在 nums1 之前）。

复制 nums2 剩余元素: 3, 2, 1
  nums1[2]=3, nums1[1]=2, nums1[0]=1

结果: [1, 2, 3, 4, 5, 6]
```

另一个例子：`nums1 = [1, 2, 3, 0, 0, 0]`（m=3），`nums2 = [2, 5, 6]`（n=3）：

```
p1=2 (值 3), p2=2 (值 6), p=5

第1步: 6 > 3 -> nums1[5]=6, p2=1, p=4
第2步: 5 > 3 -> nums1[4]=5, p2=0, p=3
第3步: 2 < 3 -> nums1[3]=3, p1=1, p=2
第4步: 2 = 2 -> nums1[2]=2（取 p1，走 else 分支）, p1=0, p=1
第5步: 2 > 1 -> nums1[1]=2, p2 移到 nums2 之前, p=0

主循环退出（p2 在 nums2 之前）。没有 nums2 剩余元素。

结果: [1, 2, 2, 3, 5, 6]
```

## 容易踩的坑

1. **从后往前合并，而不是从前往后。** 如果从前面（索引 0）开始合并，每次插入都要把 `nums1` 中剩余元素右移，把 O(m+n) 变成 O(m*n)。从后往前合并完全避免了这个问题，因为尾部的槽位是空的占位符。

2. **只有 `nums2` 需要收尾复制循环。** 主循环结束后，如果 `nums2` 还有剩余元素，必须复制过来。但如果 `nums1` 有剩余元素，它们已经在正确的位置上了（它们一开始就在那里，没有被覆盖）。

3. **用指针比较做循环边界。** 代码用 `nums1 <= p1` 和 `nums2 <= p2` 而不是基于索引的比较。当 `m=0` 时，`p1 = nums1 - 1` 在 `nums1` 之前，所以 `nums1 <= p1` 立即为假。`n=0` 同理。这样优雅地处理了边界情况，无需特殊判断。

4. **`swap` 函数未被使用。** 代码定义了 `swap`，但 `merge` 并没有调用它。合并通过直接指针赋值（`*p-- = *p2--`）完成，比交换更高效。

---

## 复杂度分析

| 指标 | 值 | 原因 |
|------|-----|------|
| **时间** | O(m + n) | 每个元素恰好被访问和放置一次 |
| **空间** | O(1) | 只用了三个指针变量；合并在 `nums1` 内原地进行 |

这已经是该问题的最优解——时间不可能比 O(m+n) 更好，因为每个元素至少要被检查一次。

---

## 总结

合并两个有序数组的方法是从后往前填充 `nums1`。三个指针分别追踪两个数组数据的末尾和写入位置。每一步将两个末尾元素中较大的放到写入位置，源指针和写入指针都向前移动。主循环结束后，将 `nums2` 的剩余元素复制过来即可。这种方法避免了从前往后合并时的元素移位问题，实现了 O(m+n) 时间和 O(1) 额外空间——已经是最优解。
