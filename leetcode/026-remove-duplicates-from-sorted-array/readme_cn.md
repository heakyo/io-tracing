## 题目描述

给定一个按**非递减顺序**排列的整数数组 `nums`，要求**原地**删除其中的重复元素，使每个元素只出现一次，并返回去重后的数组长度 `k`。去重后，`nums` 的前 `k` 个位置必须按原始顺序存放这些唯一元素；下标 `k` 之后的内容无关紧要。

所谓"原地"，是指不能分配额外数组，只能在给定的 `nums` 上面直接修改。

| 输入 | k | 前 k 个元素 |
|------|---|-------------|
| `[1, 1, 2]` | 2 | `[1, 2]` |
| `[0, 0, 1, 1, 1, 2, 2, 3, 3, 4]` | 5 | `[0, 1, 2, 3, 4]` |

---

## 核心思路

这道题用**双指针法**解决，整个想法可以用一句话概括：一根指针负责读，一根指针负责写。

- `i` 是**读指针**，从左到右逐个扫描整个数组。
- `k` 是**写指针**，始终指向下一个应该放置唯一元素的位置。换句话说，`nums[0..k-1]` 这段区间就是我们已经收集好的唯一元素前缀。

算法逻辑如下：

```c
int removeDuplicates(int* nums, int numsSize)
{
	int i, k = 1;

	for (i = 1; i < numsSize; i++) {
		if (nums[i] != nums[k-1]) {
			swap(&nums[k], &nums[i]);
			k++;
		}
	}

	return k;
}
```

对于读指针扫描到的每一个 `nums[i]`，我们把它与 `nums[k-1]`（唯一前缀中最后一个元素）比较：

- **相等**：说明是重复值，跳过，只推进 `i`。
- **不相等**：说明遇到了新值，把它 swap 到 `nums[k]` 的位置，然后 `k` 向前推进一步。

为什么只需要和 `nums[k-1]` 比较就够了？因为数组已排序，相同的值必然紧挨在一起。一旦某个值被收入唯一前缀，后续的重复值在原数组中一定是连续出现的。所以"当前读到的值"只要和"唯一前缀的最后一个"不同，就一定是一个全新的值。

## 逐步演练

### 演练一: `[1, 1, 2]`

初始状态：`k = 1`，`i = 1`。第一个元素 `nums[0] = 1` 天然属于唯一前缀，因此两个指针都从下标 1 开始。

**i = 1**：`nums[1] = 1`，`nums[k-1] = nums[0] = 1`，相等，是重复值，跳过。

**i = 2**：`nums[2] = 2`，`nums[k-1] = nums[0] = 1`，不相等，遇到新值。执行 `swap(nums[1], nums[2])`，数组变为 `[1, 2, 1]`，`k` 变为 2。

循环结束，返回 `k = 2`。前 2 个元素为 `[1, 2]`。

### 演练二: `[0, 0, 1, 1, 1, 2, 2, 3, 3, 4]`

初始状态：`k = 1`，`i = 1`。

| i | nums[i] | nums[k-1] | 相等? | 动作 | k | 数组状态（前 k 个） |
|---|---------|-----------|-------|------|---|---------------------|
| 1 | 0 | 0 | 是 | 跳过 | 1 | `[0]` |
| 2 | 1 | 0 | 否 | swap(nums[1], nums[2]), k++ | 2 | `[0, 1]` |
| 3 | 1 | 1 | 是 | 跳过 | 2 | `[0, 1]` |
| 4 | 1 | 1 | 是 | 跳过 | 2 | `[0, 1]` |
| 5 | 2 | 1 | 否 | swap(nums[2], nums[5]), k++ | 3 | `[0, 1, 2]` |
| 6 | 2 | 2 | 是 | 跳过 | 3 | `[0, 1, 2]` |
| 7 | 3 | 2 | 否 | swap(nums[3], nums[7]), k++ | 4 | `[0, 1, 2, 3]` |
| 8 | 3 | 3 | 是 | 跳过 | 4 | `[0, 1, 2, 3]` |
| 9 | 4 | 3 | 否 | swap(nums[4], nums[9]), k++ | 5 | `[0, 1, 2, 3, 4]` |

循环结束，返回 `k = 5`。前 5 个元素为 `[0, 1, 2, 3, 4]`。

## 容易踩的坑

**(a) swap 与简单赋值**

代码中用了 `swap(&nums[k], &nums[i])` 来把新值放到写指针的位置。实际上，这里用简单赋值 `nums[k] = nums[i]` 就足够了，因为 `nums[k]` 位置上的旧值后续不会再被读取（它要么是重复值，要么已经被收录过了）。`swap` 不影响正确性，但做了一次多余的写操作。两种写法在结果的前 `k` 个元素上完全一致。

**(b) k 从 1 开始，而不是 0**

`k` 的初始值为 1，`i` 也从 1 开始。这是因为数组的第一个元素 `nums[0]` 无论如何都是唯一的——它前面没有别的元素可以和它重复。所以唯一前缀天然就包含了 `nums[0]`，写指针直接从下标 1 开始等待下一个新值。

**(c) 和 `nums[k-1]` 比较，而不是 `nums[i-1]`**

初学者容易写成 `nums[i] != nums[i-1]`，即"当前元素和它的前一个邻居比较"。当 `i` 和 `k` 同步前进时，两种写法结果恰好相同，所以一开始不容易发现差异。但一旦 `k` 落后于 `i`（即跳过了若干重复值之后），`nums[i-1]` 可能是原数组中尚未被整理的旧值，而 `nums[k-1]` 才是唯一前缀的真正末尾。正确的比较对象必须是已确认的唯一前缀的最后一个元素，即 `nums[k-1]`。

不过要注意，在本题的 `swap` 实现下，由于排序数组的性质，`nums[i-1]` 写法在大多数情况下也能得到正确结果。但从语义清晰度和鲁棒性来讲，`nums[k-1]` 才是精确的表达。

**(d) 没有处理 `numsSize == 0` 的情况**

代码直接假设 `numsSize >= 1`，没有对空数组做特判。如果传入空数组，`k = 1` 会返回一个错误的结果。但在 LeetCode 的题目约束中明确写明 `1 <= nums.length <= 3 * 10^4`，所以 `numsSize` 至少为 1，这里不做检查是安全的。如果在生产代码中使用，建议增加 `if (numsSize == 0) return 0;` 的防御性判断。

## 复杂度分析

- **时间复杂度**：O(n)，其中 n 为数组长度。读指针 `i` 从 1 遍历到 `n-1`，每个元素恰好被访问一次，`swap` 操作为 O(1)，因此总时间为线性。
- **空间复杂度**：O(1)。除了几个整型变量（`i`、`k`、`tmp`）之外，没有使用任何额外空间。所有操作都在原数组上完成。

---

## 总结

这道题的本质是：在一个已排序的数组上，用一快一慢两根指针把唯一元素"捡"到数组前部。慢指针 `k` 维护着一段干净的唯一前缀，快指针 `i` 负责向前探路。每当 `i` 发现一个与唯一前缀末尾不同的新值，就把它搬到 `k` 的位置，然后 `k` 前进一步。因为数组有序，重复值一定扎堆出现，所以只要和 `nums[k-1]` 做一次比较就能判定是否为新值。整个过程只扫描一遍数组，时间 O(n)，空间 O(1)，是原地去重问题最经典、最简洁的解法。
