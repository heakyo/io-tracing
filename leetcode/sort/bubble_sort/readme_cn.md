## 题目描述

给定一个未排序的整数数组，使用冒泡排序算法将其按升序排列。

示例：

```
输入:  [1, 7, 3, 8, 4]
输出:  [1, 3, 4, 7, 8]
```

---

## 核心思路

想象一排人站成一列，每人手里举着一张数字卡片。你沿着队伍走，每次比较相邻两个人。如果前面那个人的数字比后面的大，就让他们交换位置。走到队尾时，最大的数字就"冒泡"到了最后面。

然后重复：再走一遍队伍，第二大的数字就冒泡到倒数第二个位置。一直重复，直到不需要再交换——队伍就排好了。

在代码中，两层嵌套循环完成这个过程：

- **外层循环**（`i`）记录已完成的趟数。完成 `i` 趟后，最后 `i` 个元素已经就位。
- **内层循环**（`j`）从头走到 `len - 1 - i`，比较相邻元素对，乱序则交换。

## 逐步演练

以 `[1, 7, 3, 8, 4]` 为例：

```
第0趟 i=0（把最大的冒泡到末尾）:
  j=0: [1, 7, 3, 8, 4]  1 <= 7  不交换
  j=1: [1, 7, 3, 8, 4]  7 > 3   交换 -> [1, 3, 7, 8, 4]
  j=2: [1, 3, 7, 8, 4]  7 <= 8  不交换
  j=3: [1, 3, 7, 8, 4]  8 > 4   交换 -> [1, 3, 7, 4, 8]
  本趟结束：8 就位。

第1趟 i=1（冒泡次大的）:
  j=0: [1, 3, 7, 4, 8]  1 <= 3  不交换
  j=1: [1, 3, 7, 4, 8]  3 <= 7  不交换
  j=2: [1, 3, 7, 4, 8]  7 > 4   交换 -> [1, 3, 4, 7, 8]
  本趟结束：7 就位。

第2趟 i=2:
  j=0: [1, 3, 4, 7, 8]  1 <= 3  不交换
  j=1: [1, 3, 4, 7, 8]  3 <= 4  不交换
  本趟结束：无交换，但循环继续。

第3趟 i=3:
  j=0: [1, 3, 4, 7, 8]  1 <= 3  不交换
  本趟结束：完成。

结果：[1, 3, 4, 7, 8]
```

## 容易踩的坑

1. **内层循环边界逐趟缩小。** 内层循环到 `len - 1 - i`，不是 `len - 1`。每趟过后，尾部又多一个元素已经就位。忘了 `- i` 结果仍然正确，但会浪费比较次数。

2. **没有提前终止。** 如果数组在所有趟完成之前就已经排好序（比如已排序的测试用例），这个实现仍然继续循环。它总是做完全部 `n*(n-1)/2` 次比较。优化版本增加了 `swapped` 标志来提前跳出。

3. **稳定性。** 冒泡排序是**稳定的**——相等元素保持原有的相对顺序，因为比较用的是严格的 `>`（不是 `>=`）。如果改成 `>=`，相等元素就会被不必要地交换，稳定性就丢了。

4. **外层循环的边界。** 外层循环 `i` 从 `0` 到 `len - 2`（即 `i < len - 1`）。经过 `len - 1` 趟后，只剩一个未排序的元素，单个元素天然有序。如果写成 `i < len`，最后一趟内层循环会执行零次——无害但浪费。

---

## 复杂度分析

| 指标 | 值 | 原因 |
|------|-----|------|
| **时间（最好）** | O(n^2) | 没有提前终止——即使数组已排序也要跑完所有趟 |
| **时间（平均/最坏）** | O(n^2) | 两层嵌套循环，每层最多 n 次迭代 |
| **空间** | O(1) | 只需要常数额外空间（一个临时变量用于交换） |

注意：加上提前终止优化（见 `optimized/`），最好情况可降为 O(n)。

---

## 总结

冒泡排序反复遍历数组，比较相邻元素，如果乱序就交换。每趟遍历后，下一个最大的元素就沉到了尾部的正确位置。两层嵌套循环驱动整个过程：外层计数趟数，内层做比较。所有情况下都是 O(n^2)，因为没有提前退出——即使数组已经排好序，每趟仍然照跑。算法简单、稳定、原地排序，但对大输入效率不高。优化版本在 `optimized/` 目录中增加了 `swapped` 标志，在无交换时提前退出。
