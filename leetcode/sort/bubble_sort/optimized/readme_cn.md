## 为什么要优化？

原始冒泡排序无论数组是否已经有序，总是跑完全部 `n-1` 趟。对于一个已排序的 1,000 个元素的数组，仍然要执行约 500,000 次比较——全是浪费。

优化版可以在一趟之内检测到数组已有序（O(n)），并且通过跟踪最后一次交换的位置，更高效地跳过已排序的尾部区域。

---

## 核心思路

想象同样一排举着数字卡片的人。原始版本不管怎样都要走 `n-1` 遍。优化版更聪明：

1. **如果走完一遍没有人交换，说明队伍已经排好了——立即停止。** 这就是"提前终止"的思路。

2. **记住最后一次交换发生的位置。** 那个位置之后的所有人已经有序，下次只需要走到那个位置——不用走完整个剩余长度。

在代码中，一个变量 `bound` 跟踪需要扫描的范围。每趟结束后，`bound` 更新为最后一次交换的位置（`new_bound`）。如果没有发生交换，`new_bound` 保持为 0，while 循环条件 `bound > 0` 不成立，直接退出。

## 逐步演练

以 `[1, 7, 3, 8, 4]` 为例：

```
初始状态: bound = 4

第1趟（扫描 j=0..3）:
  j=0: 1 <= 7  不交换
  j=1: 7 > 3   交换 -> [1, 3, 7, 8, 4]  new_bound=1
  j=2: 7 <= 8  不交换
  j=3: 8 > 4   交换 -> [1, 3, 7, 4, 8]  new_bound=3
  bound = 3

第2趟（扫描 j=0..2）:
  j=0: 1 <= 3  不交换
  j=1: 3 <= 7  不交换
  j=2: 7 > 4   交换 -> [1, 3, 4, 7, 8]  new_bound=2
  bound = 2

第3趟（扫描 j=0..1）:
  j=0: 1 <= 3  不交换
  j=1: 3 <= 4  不交换
  new_bound 保持为 0 -> bound = 0

bound == 0, 退出。结果：[1, 3, 4, 7, 8]
```

再看已排序的情况 `[1, 2, 3, 4, 5]`：

```
初始状态: bound = 4

第1趟（扫描 j=0..3）:
  j=0: 1 <= 2  不交换
  j=1: 2 <= 3  不交换
  j=2: 3 <= 4  不交换
  j=3: 4 <= 5  不交换
  new_bound 保持为 0 -> bound = 0

bound == 0, 只跑了一趟就退出！结果：[1, 2, 3, 4, 5]
```

原始版本这里要跑 4 趟，优化版只要 1 趟。

## 容易踩的坑

1. **`new_bound` 必须初始化为 0，不能是 `bound`。** 如果初始化为 `bound`，边界永远不会缩小。设为 0 表示"还没有交换发生"——如果保持为 0，就说明整个扫描区域已经有序。

2. **`new_bound = j`，不是 `j + 1`。** 我们记录的是交换对中左边元素的位置。交换后，`array[j+1]` 已经正确（较大值），所以下一趟只需扫描到 `j`。如果设为 `j + 1`，每趟会多做一次不必要的比较。

3. **最坏情况仍然是 O(n^2)。** 提前退出和边界收缩优化了最好情况和平均情况，但逆序输入仍然需要 `n-1` 趟，每趟都有交换。冒泡排序 O(n^2) 的本质没有改变。

4. **单元素和空数组。** 当 `len <= 1` 时，`bound = len - 1 <= 0`，while 循环不会执行。无需特殊检查就能正确处理边界情况。

## 复杂度分析

| 指标 | 值 | 原因 |
|------|-----|------|
| **时间（最好）** | O(n) | 已排序数组：一趟扫描，无交换，立即退出 |
| **时间（平均）** | O(n^2) | 随机输入中大多数趟仍然有交换 |
| **时间（最坏）** | O(n^2) | 逆序数组：每趟都有交换，`bound` 每次只缩小 1 |
| **空间** | O(1) | 除输入数组外只用了几个整数变量 |

---

## 与原始解法对比

| | 原始版本 | 优化版本 |
|--|---------|---------|
| **提前退出** | 无——总是跑完全部 n-1 趟 | 有——一趟无交换就退出 |
| **边界跟踪** | 每趟缩小 1（`len-1-i`） | 直接跳到最后交换位置——可以跳过多个元素 |
| **最好时间** | O(n^2) | O(n) |
| **最坏时间** | O(n^2) | O(n^2)（不变） |
| **循环结构** | 两层 for 循环，计数器 `i` | while 循环 + `bound` + 内层 for 循环 |

---

## 总结

优化冒泡排序在原始版本基础上增加了两个改进。第一，跟踪每趟是否发生了交换——如果没有，数组已排序，立即退出，对已排序输入实现 O(n) 最好情况。第二，不再每趟固定缩小一个位置的扫描范围，而是直接跳到最后交换的位置，跳过提前就位的尾部元素。这些改动不能改善 O(n^2) 的最坏情况，但对于部分有序或接近有序的数组能显著减少工作量——而这在实际中很常见。
