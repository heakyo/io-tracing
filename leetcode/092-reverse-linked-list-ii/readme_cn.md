## 问题描述

给定一个单链表的头节点 `head`，以及两个整数 `left` 和 `right`（1-indexed，且 `left <= right`），请你反转从位置 `left` 到位置 `right` 的链表节点，并返回修改后的链表。

**约束条件：**

- 链表节点数 `n` 满足 `1 <= n <= 500`
- `1 <= left <= right <= n`
- 节点值范围：`-500 <= Node.val <= 500`

**示例：**

```
输入: head = [1,2,3,4,5], left = 2, right = 4
输出: [1,4,3,2,5]
```

## 核心思路

想象你手里有一串珠子（链表），你需要把中间某一段的顺序翻转过来。最直观的办法是什么？——**头插法**。

用大白话讲，头插法的核心动作就一句话：

> **反复把 `cur` 后面的那颗珠子摘下来，塞到反转段的最前面。**

整个算法分三步：

### 第一步：造一个 dummy 节点

```c
dummy = malloc(sizeof(*dummy));
dummy->next = head;
```

为什么要多此一举？因为当 `left == 1` 时，链表的头节点自身要被反转，`head` 会变。有了 `dummy` 这个"假头"，不管 `left` 是几，`pre` 都有一个安全的锚点可以站。最后返回 `dummy->next` 就是新链表的真正头节点。

### 第二步：导航到起点

```c
for (i = 1; i < left; i++) {
	pre = cur;
	cur = cur->next;
}
```

让 `pre` 停在第 `left - 1` 个节点，`cur` 停在第 `left` 个节点。`pre` 是反转段的"左护栏"，它自己不参与反转，但它的 `next` 指针会不断更新，始终指向当前反转段的第一个节点。

### 第三步：头插法反转

```c
for (i = left; i < right; i++) {
	tmp = pre->next;
	pre->next = cur->next;
	cur->next = cur->next->next;
	pre->next->next = tmp;
}
```

每一轮做的事情：

1. `tmp = pre->next` — 先记住当前反转段的第一个节点
2. `pre->next = cur->next` — 把 `cur` 后面那个节点摘出来，接到 `pre` 后面（成为新的反转段头部）
3. `cur->next = cur->next->next` — `cur` 跳过被摘走的节点，连接到更后面
4. `pre->next->next = tmp` — 被摘出的节点的 `next` 指向原来的反转段头部

`cur` 自始至终不移动——它一直指向原来第 `left` 个节点。随着反转进行，`cur` 从反转段的头部逐渐"下沉"到反转段的尾部。

## 逐步推演

以 `head = [1,2,3,4,5], left = 2, right = 4` 为例，完整走一遍。

### 初始化

```
dummy -> 1 -> 2 -> 3 -> 4 -> 5 -> NULL
pre = dummy, cur = Node(1)
```

### 导航阶段（`i = 1` 到 `i < left = 2`，循环 1 次）

| 步骤 | `pre` | `cur` | 链表状态 |
|------|-------|-------|---------|
| 初始 | dummy | Node(1) | dummy -> 1 -> 2 -> 3 -> 4 -> 5 |
| i=1 | Node(1) | Node(2) | dummy -> 1 -> 2 -> 3 -> 4 -> 5 |

导航完成：`pre = Node(1)`，`cur = Node(2)`。

### 反转阶段（`i = left = 2` 到 `i < right = 4`，循环 2 次）

#### 第 1 轮（i=2）：摘出 Node(3)，插到 `pre` 后面

操作前：

```
pre          cur
 |            |
 v            v
 1  ->  2  ->  3  ->  4  ->  5
```

四步指针操作：

```
tmp = pre->next            => tmp = Node(2)
pre->next = cur->next      => pre->next = Node(3)    -- 摘出 Node(3)
cur->next = cur->next->next => cur->next = Node(4)   -- cur 跳过 Node(3)
pre->next->next = tmp      => Node(3)->next = Node(2) -- Node(3) 指向原头部
```

操作后：

```
pre               cur
 |                 |
 v                 v
 1  ->  3  ->  2  ->  4  ->  5
```

| 变量 | 指向 |
|------|------|
| `pre` | Node(1) |
| `cur` | Node(2)（没动，但已下沉一位） |
| `pre->next` | Node(3)（新的反转段头部） |

#### 第 2 轮（i=3）：摘出 Node(4)，插到 `pre` 后面

操作前：

```
pre               cur
 |                 |
 v                 v
 1  ->  3  ->  2  ->  4  ->  5
```

四步指针操作：

```
tmp = pre->next            => tmp = Node(3)
pre->next = cur->next      => pre->next = Node(4)    -- 摘出 Node(4)
cur->next = cur->next->next => cur->next = Node(5)   -- cur 跳过 Node(4)
pre->next->next = tmp      => Node(4)->next = Node(3) -- Node(4) 指向原头部
```

操作后：

```
pre                    cur
 |                      |
 v                      v
 1  ->  4  ->  3  ->  2  ->  5
```

| 变量 | 指向 |
|------|------|
| `pre` | Node(1) |
| `cur` | Node(2)（下沉到反转段尾部） |
| `pre->next` | Node(4)（新的反转段头部） |

### 最终结果

```
返回 dummy->next: 1 -> 4 -> 3 -> 2 -> 5
```

### 完整变化总结

| 轮次 | 摘出节点 | 链表状态 |
|------|---------|---------|
| 导航后 | — | 1 -> 2 -> 3 -> 4 -> 5 |
| 第 1 轮 (i=2) | Node(3) | 1 -> **3 -> 2** -> 4 -> 5 |
| 第 2 轮 (i=3) | Node(4) | 1 -> **4 -> 3 -> 2** -> 5 |

## 易错点分析

### (a) 为什么需要 dummy 节点？

当 `left == 1` 时，链表的第一个节点就在反转范围内，反转完成后 `head` 指向的节点已经不再是链表头部了。如果没有 `dummy`，你就必须单独处理这个边界情况。

`dummy` 提供了一个**稳定的锚点**——不管反转区间怎么变，`dummy->next` 永远指向最新的链表头节点。这就是"哨兵节点"的经典用法。

### (b) cur 为什么不移动？

这是最反直觉的地方。`cur` 始终指向**原来**第 `left` 个节点。反转过程中，`cur` 前面不断被塞入新节点，所以 `cur` 自然而然地"下沉"到了反转段的末尾。

不需要移动 `cur`，恰恰是头插法的精妙之处：`cur` 既是"操作手"（从它后面摘节点），又自动成为反转段的尾巴。

### (c) 内存泄漏

```c
dummy = malloc(sizeof(*dummy));
// ... 使用 dummy ...
return dummy->next;  // dummy 本身从未 free！
```

`dummy` 是 `malloc` 分配的堆内存，但函数返回时直接返回了 `dummy->next`，`dummy` 自身的内存再也找不回来了。

LeetCode 的评测系统不检查内存泄漏，所以能通过。但在生产代码中，应该这样修复：

```c
struct ListNode *result = dummy->next;
free(dummy);
return result;
```

### (d) 四行指针操作的顺序至关重要

反转循环中的四行代码：

```c
tmp = pre->next;          // 1. 先保存
pre->next = cur->next;    // 2. 摘出节点
cur->next = cur->next->next; // 3. cur 跳过
pre->next->next = tmp;    // 4. 拼接
```

顺序不能乱。特别是第 1 行 `tmp = pre->next` 必须在最前面，因为第 2 行会覆盖 `pre->next`。如果不先保存，原来的 `pre->next` 就丢了，链表就断了。

同样，第 3 行必须在第 2 行之后：第 2 行执行后 `pre->next` 已经指向了 `cur->next`（被摘出的节点），第 3 行中的 `cur->next->next` 实际上是通过原始链接访问到被摘出节点后面的节点。如果先执行第 3 行，`cur->next` 还没被摘出，`cur->next->next` 的语义就不对了。

## 复杂度分析

| 维度 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | **O(n)** | 导航阶段最多走 `left - 1` 步，反转阶段走 `right - left` 步，总计最多 `right - 1` 步，不超过 `n` |
| 空间 | **O(1)** | 只使用了 `dummy`、`pre`、`cur`、`tmp` 四个指针变量。`dummy` 的 `malloc` 是 O(1) 常数空间（虽然存在泄漏） |

---

## 总结

头插法反转子链表的核心思想可以浓缩为一句话：

> **`cur` 不动，反复把 `cur` 后面的节点摘下来，插到反转段最前面（`pre` 后面）。**

这个算法的优雅之处在于：

1. **单次遍历**：不需要先找到反转段再整段反转，边走边翻，一趟搞定。
2. **`cur` 的双重角色**：既是摘取节点的操作起点，又自然下沉成为反转段的尾节点。
3. **`dummy` 的统一性**：用一个哨兵节点抹平了 `left == 1` 的边界问题，代码无需分支处理。

记住这个画面：`pre` 是左护栏，`cur` 是传送带的起点。每一轮，传送带把 `cur` 右边的节点搬到 `pre` 的右边。搬完 `right - left` 次，反转就完成了。
