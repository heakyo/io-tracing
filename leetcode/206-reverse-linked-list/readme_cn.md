## 问题描述

给定单链表的头节点 `head`，反转链表并返回反转后的链表。

**约束条件：**

- 链表中节点的数目范围是 `[0, 5000]`
- `-5000 <= Node.val <= 5000`

**示例：**

```
输入：head = [1, 2, 3, 4, 5]
输出：[5, 4, 3, 2, 1]
```

---

## 核心思路

> 想象你面前有一串珠子穿在绳子上（正向链表），你要把它们一颗一颗摘下来，
> 然后按相反的顺序重新穿到另一根绳子上（反转链）。

算法采用**迭代原地反转**策略。逐节点遍历链表，将每个节点从正向链中摘下，
插入到反转链（`rh`）的头部。用 `head` 作为临时变量保存下一个节点，避免覆盖
`p->next` 后丢失后续链表。循环结束后 `rh` 指向新的头节点。

三个指针各司其职：

| 指针 | 角色 | 初始值 |
|------|------|--------|
| `rh` | 反转后的头节点（已反转部分的链头） | `NULL` |
| `p` | 游标（当前正在处理的节点） | `head` |
| `head` | 临时变量（保存下一个待处理节点） | 原始头节点，随后被复用 |

核心代码如下：

```c
struct ListNode* reverseList(struct ListNode* head)
{
	struct ListNode *rh = NULL, *p = head;

	while (p) {
		head = head->next;   /* 保存下一个节点 */
		p->next = rh;        /* 当前节点指向反转链头部 */
		rh = p;              /* 更新反转链头部 */
		p = head;            /* 移动游标到下一个节点 */
	}

	return rh;
}
```

用费曼的话说：**每一轮循环做的事情就是"摘一颗珠子，穿到新绳子最前面"。**

## 逐步推演

以 `[1, 2, 3, 4, 5]` 为例，完整追踪每一轮循环中三个指针的变化：

| 轮次 | 操作前 `p` | `head = head->next` | `p->next = rh` | `rh = p` | `p = head` | 反转链状态 |
|------|-----------|---------------------|-----------------|----------|------------|-----------|
| 初始 | — | — | — | — | — | `rh=NULL, p=1->2->3->4->5` |
| 第 1 轮 | `1` | `head=2->3->4->5` | `1->NULL` | `rh=1->NULL` | `p=2` | `1->NULL` |
| 第 2 轮 | `2` | `head=3->4->5` | `2->1->NULL` | `rh=2->1->NULL` | `p=3` | `2->1->NULL` |
| 第 3 轮 | `3` | `head=4->5` | `3->2->1->NULL` | `rh=3->2->1->NULL` | `p=4` | `3->2->1->NULL` |
| 第 4 轮 | `4` | `head=5` | `4->3->2->1->NULL` | `rh=4->3->2->1->NULL` | `p=5` | `4->3->2->1->NULL` |
| 第 5 轮 | `5` | `head=NULL` | `5->4->3->2->1->NULL` | `rh=5->4->3->2->1->NULL` | `p=NULL` | `5->4->3->2->1->NULL` |

循环结束，`p=NULL`，返回 `rh`：`5->4->3->2->1->NULL` ✓

## 易错点分析

### (a) 复用 `head` 作为临时变量

参数 `head` 被重新用于保存 `next` 指针，这种做法不常见。大多数实现会使用
一个显式的 `next` 变量，语义更清晰：

```c
/* 更常见的写法 */
struct ListNode *next = p->next;  /* 显式命名，意图一目了然 */
p->next = rh;
rh = p;
p = next;
```

复用 `head` 虽然节省了一个局部变量声明，但增加了阅读者的心智负担——`head`
在函数入口表示"链表头"，进入循环后语义悄然变成了"下一个待处理节点"。

### (b) 空链表处理

当 `head` 为 `NULL` 时：

- `p = head`，所以 `p = NULL`
- `while (p)` 条件不满足，循环不执行
- 直接返回 `rh = NULL`

结果正确，无需额外的边界判断。

### (c) 操作顺序至关重要

循环体内四条语句的顺序**不能随意调换**。必须先保存 `head->next`，再覆盖
`p->next`，否则会丢失后续链表。如果先执行 `p->next = rh`，那么 `head->next`
（此时 `head` 和 `p` 指向同一节点）就已经被修改，后续链表从此丢失。

```
正确顺序：保存 next → 断开当前节点 → 接入反转链 → 前进游标
错误顺序：断开当前节点 → 保存 next（已被覆盖！）→ 💥
```

## 复杂度分析

| 维度 | 复杂度 | 说明 |
|------|--------|------|
| **时间** | O(n) | 遍历链表一次，每个节点恰好处理一次 |
| **空间** | O(1) | 仅使用 `rh`、`p`、`head` 三个指针变量，无额外内存分配 |

---

## 总结

算法通过维护一条反转链（`rh`）并将每个节点插入其头部来实现**单次遍历的原地
反转**。每一轮循环做的事情可以用一句话概括：

> **摘下当前节点，插到反转链最前面，然后前进一步。**

复用 `head` 参数作为临时变量保存 `next` 指针，功能完全正确，但可读性不如
使用显式的 `next` 变量。理解这一点后，整个算法就像把一叠扑克牌逐张翻面
重新叠放一样简单。
