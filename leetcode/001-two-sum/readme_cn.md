## 题目描述

给定一个整数数组 `nums` 和一个整数 `target`，找出数组中 **两个数的下标**，使它们的和等于 `target`。

可以假设每个输入 **恰好有一个解**，且同一个元素不能使用两次。

示例：

```
输入:  nums = [2, 7, 11, 15], target = 9
输出:  [0, 1]
解释:  nums[0] + nums[1] = 2 + 7 = 9
```

---

## 核心思路

想象你站在一个房间里，每个人手里举着一张数字卡片。有人喊出一个目标数，你要找到 **两个人**，他们卡片上的数字之和等于目标数。

最简单的办法（暴力法）是：逐个找每一个人，然后依次看其他所有人，检查两张卡片加起来是否等于目标。代码正是这么做的 —— **两层嵌套循环**，尝试每一种配对。

外层循环选一个数，内层循环扫描它后面的所有数。一旦找到匹配的一对，立即返回它们的下标。

## 逐步演练

以 `nums = [3, 2, 3]`，`target = 6` 为例：

```
第1步: i=0, 选 nums[0]=3
       j=1: nums[0]+nums[1] = 3+2 = 5 != 6  -> 跳过
       j=2: nums[0]+nums[2] = 3+3 = 6 == 6  -> 找到了!
       返回 [0, 2]
```

另一个例子：`nums = [2, 7, 11, 15]`，`target = 9`：

```
第1步: i=0, 选 nums[0]=2
       j=1: nums[0]+nums[1] = 2+7 = 9 == 9  -> 找到了!
       返回 [0, 1]
```

再来一个：`nums = [3, 2, 4]`，`target = 6`：

```
第1步: i=0, 选 nums[0]=3
       j=1: 3+2 = 5 != 6  -> 跳过
       j=2: 3+4 = 7 != 6  -> 跳过

第2步: i=1, 选 nums[1]=2
       j=2: 2+4 = 6 == 6  -> 找到了!
       返回 [1, 2]
```

---

## 容易踩的坑

1. **不能重复使用同一个元素。** 内层循环从 `j = i + 1` 开始，而不是 `j = 0`，这样就不会把一个元素和自己配对。

2. **提前返回。** 找到一对就立即返回。题目保证恰好有一个解，所以不需要继续搜索。

3. **找不到解的情况。** 如果没有配对成功（按题目约束不会出现），函数返回 `[0, 0]` —— 即 memset 初始化的默认值。在实际工程中，这种边界情况应该更明确地处理。

4. **变量 `k` 的作用。** 代码中 `k = i` 然后用 `nums[k]` 和 `k + 1`，其实完全等价于直接使用 `i`。这不会引入 bug，但属于不必要的间接引用。

---

## 复杂度分析

| 指标 | 值 | 原因 |
|------|-----|------|
| **时间** | O(n^2) | 两层嵌套循环，每层最多遍历 n 个元素 |
| **空间** | O(1) | 只分配了一个固定大小（2个元素）的结果数组 |

这种暴力解法最容易理解。更优的做法是使用 **哈希表** 实现 O(n) 时间 —— 遍历数组时，对于每个数，检查 `target - nums[i]` 是否已经在哈希表中；如果不在，就把当前数及其下标存入哈希表。

---

## 总结

两数之和是经典的"找配对"问题。这里的暴力解法用两层嵌套循环尝试所有 (i, j) 配对，找到和等于 target 的那一对就返回。时间复杂度 O(n^2)，空间复杂度 O(1) —— 简单正确，只是不够快。关键点在于内层循环从 `j = i + 1` 开始以避免元素和自身配对，以及找到后立即返回。
